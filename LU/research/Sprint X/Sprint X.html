Sprint X

In mijn sprint X project heb ik veel focus gelegt op development. Mijn project was het maken van een tower defense game. Hierbij heb ik gebruik gemaakt van pure HTML, CSS en javascript. Zelf had ik al ervaring met game development, maar nooit heb ik hiervoor javascript gebruikt. Verder heb ik ook research gedaan naar het maken van een javascript game.

Development / Research

Ideevorming

Voor mijn project heb ik redelijk snel de keuze gemaakt om een game te maken. Ik heb onderzoek gedaan naar verschillende soorten games door te kijken naar andere javascript games of games in bijvoorbeeld de Play store. Uiteindelijk heb ik 3 ideeÃ«n gevormd: een tower defense game, een clicker game of een kaartspel. Uiteindelijk leek de tower defense game mij het uitdagendst, dus heb ik dit idee uitgewerkt. Ik heb gekeken naar andere TD games als Bloons TD om inspiratie op te doen. Na een stappenplan en planning te maken ben ik begonnen met coderen.

javascript

Toen ik begon met het maken van de game twijfelde ik om een library te gebruiken om collision te detecteren. Uiteindelijk heb ik de keuze gemaakt om dit niet te doen. Het javascript bestand bestaat uit meerdere functies om bepaalde acties uit te voeren. In dit project heb ik gebruik gemaakt van termen waarmee ik nog niet eerder heb gewerkt in javascript. Voorbeelden hiervan zijn appendChild(), getComputedStyle() of setAttribute(). Dit heeft mijn javascript kennis in het algemeen groter gemaakt. Verder heb ik de stijl en layout gemaakt met HTML en CSS, maar hier is een stuk minder tijd in gegaan.

Feedback

Als project begeleider voor sprint X heb ik Geert Jan gevraagd. Deze keuze heb ik gemaakt omdat hij veel kennis heeft van development en omdat ik het idee had dat communicatie makkelijk zou zijn. Ik heb regelmatig feedback gevraagd op mijn proces richting het eindproduct. Deze feedback heb ik verwerkt in mijn project. Zo attendeerde Geert Jan er mij bijvoorbeeld op dat ik nog niet had gekeken naar het proces van het aanvallen van de enemies. Later bleek dat dit een redelijk groot deel van het project was. Uiteindelijk heb ik ook samen met mijn project begeleider de keuze gemaakt om een deel van de game niet uit te werken, omdat het anders niet haalbaar was.

Wat heb ik geleerd

Ik heb meer ervaring opgedaan met het onderzoeken van een onderwerp en het vormen van een idee. Verder heb ik mijn javascript vaardigheden erg verbeterd door het coderen van deze game.

Wat kon beter

Ik had eerder kunnen beseffen dat bepaalde delen van de game niet haalbaar waren in de tijd die ik had. Zo had ik mijn planning hierop kunnen aanpassen.
